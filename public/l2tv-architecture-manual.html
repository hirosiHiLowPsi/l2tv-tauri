L2TV 構造説明書

対象バージョン: 2.1.0 / 更新日: 2026-07-13
この文書は現在の実装（server.js / database-worker.js / electron-main.js / public/app.js）をもとに、機能とコード構造を把握しやすくまとめたものです。

1. 全体像

L2TV は「Node.js サーバー + フロントエンド + Electron ラッパー」で構成されます。 ブラウザ版は node server.js で起動し、デスクトップ版は Electron が内部で同サーバーを起動して表示します。 

Browser/Electron UI (public/index.html + public/app.js + public/styles.css)
        ↓ fetch("/api/*")
Node HTTP Server (server.js)
        ├─ 難易度表 (table.html → header.json → score.json)
        └─ Database Worker (score.db / song.db / Rival DB, read-only sqlite)

2. ディレクトリ構成（主要部分）

3. フロントエンド（public/app.js）

3.1 役割

  • 入力UI（score.db / song.db / 難易度表URL / テーマ / プリセット）の管理
  • API呼び出し（/api/analyze など）
  • 集計表示（Player Data / Lamp Updates / レベルグラフ / 譜面一覧）
  • IndexedDB への状態保存と復元
  • 差分比較（前回→今回）による更新ランプ抽出と画像出力

3.2 主要状態

変数内容
latestAnalysis最新解析結果全体
levelChartModelamp / score のグラフモード
selectedThemeModel2tv-pop / lr2ir-dark
selectedTablePresetIdsプリセット難易度表の選択状態
tableInfoPanelOpenState各難易度表の開閉状態
chartDetailsOpenState譜面一覧 details の開閉状態
chartListOpenLevelsStateレベル単位折りたたみ状態
latestLampImprovements前回比で更新された譜面リスト

3.3 永続化キー（IndexedDB）

  • FORM_STATE_KEY: 入力欄・テーマ・モードなど
  • LAST_ANALYSIS_KEY: 最新解析結果
  • PLAYER_PROFILE_CACHE_KEY: 取得済みプレイヤープロフィール
  • TABLE_PRESET_SELECTION_KEY: プリセット選択状態

4. サーバー（server.js）

4.1 API エンドポイント

Method/Path用途
GET /api/healthヘルスチェック
GET /api/client-configローカルAPI用トークン取得
POST /api/analyze総合解析（難易度表 + ローカルDB 突合）
POST /api/profile-from-dbscore.db から名前/段位取得
POST /api/local-db-statescore.db / song.db の更新状態（size/mtime）取得
GET /*public/ の静的配信

4.2 解析の分岐

  • score.db は必須。ローカルDB中心でランプ/スコア/BP/プレイ回数を取得
  • song.db がある場合は、曲カタログ、NO SONG 判定、st/sl段位情報を取得
  • 難易度表未入力: プレイヤーデータのみ表示

4.3 主要キャッシュ

  • responseCache: URL取得結果
  • tableCache: 読み込んだ難易度表
  • playerMyListCache: score.db 読み込み結果
  • songDbCatalogCache: song.db の曲カタログ参照結果

5. データソースと突合ロジック

5.1 難易度表

  1. table.html から <meta name="bmstable"> を取得
  2. header.json の data_url から score.json 取得
  3. 譜面を正規化（title/artist/level/md5/sha256/ヒント）

5.2 ローカルDB

  • score.db player: 名前、grade_7/grade_14、通算打鍵情報
  • score.db score: clear/playcount/perfect/great/bad/poor/minbp など
  • song.db song: hash別の曲存在判定
  • song.db grade: st/sl スキルアナライザー段位解析

6. Electron 連携

6.1 electron-main.js

  • アプリ起動時に 127.0.0.1:4173 を自プロセスで確保し、起動成功を確認してから組み込み画面を表示
  • Stellaverse IR通信は利用者が明示的に許可した場合だけ実行し、アクセスを直列化・間隔制御・キャッシュ
  • 配布版は原則 exe 隣にポータブルデータフォルダを作成して保存
  • IPC: lr2ir:pick-file（db参照ダイアログ）、 lr2ir:save-image（PNG保存）

6.2 electron-preload.js

レンダラへ window.lr2irDesktop を公開し、 pickFile / saveImage を安全に呼べるようにしています。
